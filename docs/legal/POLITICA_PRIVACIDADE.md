# Politica de Privacidade

> Modelo sujeito a revisao obrigatoria por advogado/consultor de proteccao de dados antes de uso comercial.

## Dados tratados

O Kwanza Folha pode tratar dados de empresa, utilizadores, funcionários, documentos de identificação, NIF, INSS, dados bancários, assiduidade, salários, descontos, recibos, relatórios e auditoria.

## Finalidades

Processamento salarial, cumprimento fiscal/laboral, emissao de recibos, relatórios, backups, auditoria, licenciamento e suporte.

## Responsabilidade

O cliente e normalmente o responsavel pelo tratamento dos dados que introduz. O fornecedor pode atuar como prestador de suporte, conforme contrato.

## Armazenamento

A aplicação e local/offline por desenho. Os dados ficam no computador/ambiente do cliente, salvo funcionalidades futuras ou suporte contratado.

## Seguranca

O sistema usa credenciais, hashes de palavra-passe, controlo de permissoes, auditoria e proteccao de dados locais quando configurada. O cliente deve proteger o computador, acessos e backups.

## Partilha de dados

Dados podem ser exportados pelo cliente para bancos, contabilistas, autoridades, auditores ou outros destinatarios sob responsabilidade do cliente.

## Retencao

O prazo de conservacao deve seguir requisitos legais, fiscais, laborais e politica interna do cliente.

## Direitos dos titulares

Pedidos de acesso, correção, oposicao ou eliminação devem ser avaliados pelo cliente conforme a lei aplicável e obrigacoes legais de conservacao.

## Backups

Backups podem conter dados sensiveis e devem ser cifrados, controlados e armazenados em local seguro.
